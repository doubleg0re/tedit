# Project docs

Project planning and historical design notes are grouped here so the repository root stays focused on the shipped package.

- `principles/` — product vision and design principles.
- `issues/` — implementation issue notes and follow-up proposals.
- `rfcs/` — larger design proposals.
- `plans/` — dated plans and roadmaps.
- `reviews/` — retrospective review notes.
- `dogfood/` — dogfood reports and usage observations.
- `agent-setup.md` — MCP setup guidance for agents.
- `share-local-package.md` — local package sharing guide.
- `RELEASE-PREP-2026-06-25.md` — minimal release-readiness checklist for 0.1.0.
